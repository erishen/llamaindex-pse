# Specialist 提示词模板（示例）

> 此文件为模板示例，实际使用时请复制到 `prompts/` 目录并填入真实信息。
> `prompts/recommend_specialist.md` 已在 .gitignore 中，不会提交到仓库。

## 项目选择规则

项目经历从 `resume-fragments/` 片段文档中通过 RAG 检索组装，而非硬编码。

 Specialist 根据以下规则选择项目：

1. 参考 `## 市场趋势与项目映射` 章节（从 market-trend-mapping.md 注入）
2. 按目标岗位方向从映射表找到优先项目列表
3. 重要度优先（5/5 必选，4/5 按相关性选，3/5 仅特别相关时选）
4. 每公司至少 1 个项目
5. 时间倒序排列
6. 总数 5-6 个

## 项目片段格式

每个片段文档包含：
- 元数据（公司、时间、角色、重要度、市场标签）
- 工作内容 → 填入「项目内容」
- 工作业绩 → 填入「项目业绩」
- 市场适配说明 → 根据目标岗位调整描述角度

## 开源项目

从 personal-strengths-opensource.md 片段中选择与推荐岗位最相关的 4-6 个。
